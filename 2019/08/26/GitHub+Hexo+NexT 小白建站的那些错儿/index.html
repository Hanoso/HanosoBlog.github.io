    Issue5：LeanCloud设置后Counter类前端返回403错误【close】

    👉 ==暂未找到解决办法==，如果你有办法，麻烦留言或首页E-Mail我哦^两分钟前^
    👉两分钟后：==貌似正常了==。折腾过程如下：

    • 在LeanCloud > Counter类 > 设置 Counter 的权限，将前三个的权限都改为“所有用户”或逐一改为所有用户（其他还是指定用户），刷新控制台，再刷新网站，看看是否显示，应该会正常显示；
    • 害怕漏洞，然后再按LEAFERx的方法设置回去，刷新控制台，再刷新网站，阅读次数仍然正常显示，且数量会随着刷新增加。
    • 好像就这样解决了啊。。怪不怪。。。

    Ps: 我为什么折腾上面两步？因为我的Blog里用了Valine评论插件，而且也用LeanCloud管理，这个hexo d后立马就可以用了，LeanCloud管理也正常，然后我就对Comment类的的权限设置（前三个）进行调整，看看对评论功能有啥影响，发现Create设置为指定用户时，网站评论功能会提示访问被api域名白名单拒绝之类的（还有把add_fields改为指定用户后无法回复评论，提示相同），改为所有用户则可正常评论。又看了下官方403建议。。啊啊，，，然后就对Counter类进行逐一改变权限设置的试验，看看网站文章浏览计数功能是不是正常了，==没想到就这样成了==（好像是圣经里的一句话）。。然后再按修复方法改回设置，仍然正常计数。。。==没想到就这样成了==（好像是圣经里的一句话） 哈哈

    Issue7：版权声明设置【close】

    网上能搜到很多介绍，但方法基本一样，啰嗦。可能与next版本有关，我的V3.9版的在主题配置里改下设置就行，如下：

    creative_commons:
  license: by-nc-sa
  sidebar: true  # 默认false，改为true后social link下出现版权图标
  post: true  # 默认false，改为true即显示版权信息
  language:

    效果见文章底部。

    Issue8：首页文章阴影（NextV3.9）【close】

    网上搜到的都是老代码，你会发现根本找不到_custom.styl这个文件，我F12了半天，找到了css源码位置\themes\next\source\css\_common\components\post\post.styl，定位到如下代码：

    .use-motion {  
    if (hexo-config('motion.transition.post_block')) {    
        .post-block, .pagination, .comments {      
            opacity: 0;    
        }    
    }

    对==.post-block==单独增加代码：

    .use-motion {  
    if (hexo-config('motion.transition.post_block')) {    
        .post-block, .pagination, .comments {      
            opacity: 0;    
        }    
        .post-block{      
            margin-top: 60px;      
            margin-bottom: 60px;      
            padding: 25px;      
            -webkit-box-shadow: 0 0 5px rgba(202, 203, 203, .5);      
            -moz-box-shadow: 0 0 5px rgba(202, 203, 204, .5);    
        }  
    }

    你会发现首页文章已显示阴影。

    Issue10：Chrome提示“Your connection is not private” 【close half】

    经试验，国内IP访问国外服务器地址时，谷歌浏览器会报此错，而百度简单搜索只提示证书过期但仍可继续访问。

    国外IP访问国外服务器时谷歌不会报此错。但是，虽然hexo同时push到coding和GitHub，绑定的同一域名（阿里云）解析设置也分别对应默认和海外，但关掉vpn后访问网站谷歌仍然提示“您的连接不是私密连接”。

    不带www的域名关掉和开着vpn都能正常访问了，原因应该是DNS解析设置的不对，尝试了很多种组合才成功。带www的域名，关掉VPN能正常访问，打开VPN则仍提示“不是私密连接”，这个不管了，有一个域名正常就行。

    Issue11：同时部署到Coding和GitHub【close】

    网上搜到的大多已过时，还是coding.me的地址，已不能用，容易让小白误会，比如我，穷折腾好久。

    GitHub的部署就不说了，一搜一个准。

    Coding.net的部署之坑，填平如下：

    # Deployment
## Docs: https://hexo.io/docs/deployment.html
deploy:
  - type: git
    repository: https://github.com/Hanoso/HanosoBlog.github.io.git
    branch: master
  - type: leancloud_counter_security_sync
  - type: git
    repository: https://e.coding.net/sea-north2south/HanosoBlog.git # coding.net
    branch: master

    hexo站配置_config.yml中的deploy设置如上。其中coding.net地址获得路径为： >> 代码 >> 代码浏览 ：右上角HTTPS后的链接即使deploy中repository所需地址，直接点右侧复制按钮即可。

    后续hexo d可同步部署到coding和GitHub上。此问题解决后，Issue9亦能得到解决，因为coding上有代码了啊。
